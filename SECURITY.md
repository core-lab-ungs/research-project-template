# Security Policy

## Reporting a Vulnerability

Do not disclose security vulnerabilities through public issues, discussions, or pull requests.

Report vulnerabilities privately to the project maintainers using the institutional contact identified in the repository.

Include, when possible:

- affected component and version;
- steps required to reproduce the issue;
- expected and observed behavior;
- potential security impact;
- suggested mitigation, if available.

Do not include real credentials, private keys, personal information, or confidential data in the report.

## Supported Versions

Each project must identify the versions that currently receive security updates.

| Version | Supported |
|---|---|
| Latest release | Yes |
| Older releases | Project-dependent |

## Responsible Disclosure

Maintainers should acknowledge the report, evaluate its impact, prepare a correction, and coordinate disclosure before publishing technical details.

This template must be adapted to the security requirements and communication channels of each project.
